Machine Learning Using Automated Hyperparameter Optimization and Scoring Metrics for Classifying Cheese by Fat Content¶

by Batool Azim¶

August 2023¶

Introduction¶

The cheese market in Canada is substantial, with reported revenues of 7.3 billion US dollars for this year alone (Statista, 2023). The upcoming years are projected to exhibit an annual growth rate of 7.97% until 2028.

Cheese is consumed in all provinces of Canada and is part of the Canadian population’s diet. Canada’s food guide promotes choosing lower-fat cheeses among milk and dairy products of protein foods. Additionally, Cheese.com identifies a total of 1,830 specialty cheeses. The extensive variety and diverse categories of cheeses render the fat classification process quite challenging. This is where a machine learning model can step in to facilitate the process.

The purpose of this paper is to develop a model that reduces instances where cheeses with higher fat content are incorrectly classified as lower fat. This focus is particularly aimed at individuals who monitor their saturated fat intake, aiming to keep it minimal. Misclassifying a higher fat cheese as lower fat holds greater drawbacks for this specific population. The classification model I intend to create for this project will effectively differentiate between "higher fat" as a positive label and "lower fat" as a negative label.

A greater number of features can potentially lead to a more predictive model; therefore, all the remaining features are included in my model. I use seven features to predict whether a cheese is “lower fat” or “higher fat.” The features are manufacturing province, manufacturing type, moisture percent, organic, category type, milk type, and milk treatment type.

  • Manufacturing province: Eight categories.
  • Manufacturing type: Three categories—industrial, farmstead, and artisan.
  • Moisture percent: Eight null values.
  • Organic type: Two categories—nonorganic represented by “0” and organic represented by “1.”
  • Category type: Six categories. 17 null values.
  • Milk type: Eight categories. One null value.
  • Milk treatment type: Three categories—raw milk, pasteurized, and thermised. 45 null values.

Farmstead cheese utilizes milk that was produced in the same farm as the herds that produced the milk (Cheese Origin, 2022). Artisanal cheese utilizes milk from several herds in close geographic proximity.

Moisture percent is a numerical feature so addressing the null values can be with imputation using “median” strategy. Category type, milk type, and milk treatment type are categorical features; imputation using “most_frequent” strategy can address the null values. This is shown in detail in the next section: “Methods and Results.”

There are two pipelines: “numeric_transformer” and “ctg_transformer” for the numeric and categorical features, respectively. The numeric transformer includes two actions for the numeric data: SimpleImputer and StandardScaler. The first action addresses the null values by replacing them with the median of the column. The second action standardizes the data with a sample mean of 0 and standard deviation of 1. The categorical transformer includes two actions for the categorical data: SimpleImputer and OneHotEncoder. The first action addresses the null values by replacing them with the most_frequent value in the column. The second action transforms each categorical column into x binary columns; x is the number of categories in that column.

This involves automated hyperparameter tuning for a Decision Tree Classifier using RandomizedSearchCV, which is computationally less expensive than GridSearchCV. As "min_samples_split" increases, the model's complexity decreases, and as "max_depth" increases, the model's complexity rises. If "min_samples_split" is set to 2, the model might become excessively tailored to the training dataset, leading to poor generalization on the test dataset. In simpler terms, while the training score could be high, the test score could remain low. Similarly, when "max_depth" is left at its default value of None, it results in the aforementioned undesirable outcome. After several trial-and-error attempts, I noticed no significant enhancement in the best score when reducing "min_samples_split" below 10 or increasing "max_depth" beyond 20. The difference between the training score and the cross-validation score was 20.77% before hyperparameter tuning and has now reduced to 9.91%.

Discussion¶

The above confusion matrix shows that there are 91 true positives, 170 true negatives, nine false negatives, and 43 false positives. The top left quadrant represents TP, top right quadrant represents FN, bottom left quadrant TN, and bottom right quadrant FP. The top row is of particular interest in this project as the goal is to minimize higher fat cheeses wrongly classified as lower fat.

I chose hyperparameter tuning based on recall score for the purpose of my project. The model’s (recall) score is 0.9100 on the test set. The equation to manually calculate this is shown below. There are nine false negatives and 100 actual positives, which is derived from the addition of 91 and nine. The macro-average, calculated by assigning equal weight to “higher fat” and “lower fat,” is 0.8541.

Figure 5. Recall Equation. From “Module 7,” by University of British Columbia.

The accuracy metric calculates the ratio of correct predictions to total predictions. The total correct predictions include TP and TN, and the total predictions in this project amount to 313. 261 divided by 313 is 0.8339. Another method is using accuracy_score function as shown below. The dummy score is 0.5144 and the accuracy score of the model on the test set is 0.8339.

0.8339

The model’s score of 0.91 surpasses my anticipated value of 0.80 by 11%. A few intriguing modifications for future iterations of this project involve addressing the data imbalance using an alternate approach: a) specifying “stratify” argument in the “train_test_split” function or b) using “StratifiedKFold” function for cross-validation. Furthermore, utilizing Random Forest Classifier model may yield good results. It might be worth considering imputing the dropped columns to assess their impact on the final test score. The model performed exceptionally well on the test set and could be deployed. This suggests that decision trees might be effective in handling mildly imbalanced datasets.
